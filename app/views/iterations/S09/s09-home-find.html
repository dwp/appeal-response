A new home

Hypothesis / Experiment

  • We have seen that that showing the list of previous responses gave an impression that this service was a workflow management tool
  • We know this because in the last rounds of usability testing, users could not identify what this tool's purpose was on the first impressions
  • So if we create a simpler home page with bare bones content
    and just a simple UI to upload the bundle
  • We will see a better orientation on what this service is meant for
    less distractions for users
  • We'll know this is true when users will be able to identify the purpose of the service without having to read through the text or going through the service

What's new or changed?

  • In the previous iteration we tested a home page with the list of previously created appeal responses either in draft or completed status. In this iterations we have designed a simplified home page with just the ability to upload a bundle.
  • A separate search page to search for previously created appeal responses.

What did we learn?

  • In the previous sprint we mentioned ‘orientation’ issues relating to the ‘Appeal Response’ screens. These were less evident in this round of testing. Possible reasons (i) prior knowledge of prototype (ii) new “upload evidence” concept.
  • There were ‘orientation’ findings relating to first screen users saw – including perception that it was the start of building the evidence bundle, or the first step in creating an appeal response, or compiling a response but with some form of response already in place.​

Next steps

  • Since the evidence bundle might not have been created at the point of creating a response, we will have to extract that information from a different system. We will continue evolving this simpler version of the home page and search page but might revert back to creating the response by extrating data from HMCTS' CCD system.
